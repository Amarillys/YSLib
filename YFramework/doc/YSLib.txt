@1 Shell 消息列表：

//全局系统消息。
//除另有说明外，一般用作队列消息。

//空消息。参考优先级 0xFF 。
//参数类型 void 。
//通常除了对消息计数外，应被无条件忽略。
SM_NULL					0x0000

// Shell 设置消息。参考优先级 0x80 。
//参数类型 shared_ptr<Shell>。
//指示线程控制权移交给目标 Shell 。参数是目标 Shell 句柄。
SM_SET					0x0003

// Shell 激活消息。参考优先级 0xF0 。
//参数类型 void 。
// Shell 被激活后自动产生。默认为非队列消息。
SM_ACTIVATED			0x0005

// Shell 停用消息。参考优先级 0xF0 。
//参数类型 void 。
//Shell 被取消激活状态后自动产生。默认为非队列消息。转移控制权时，总在 SM_ACTIVATED 之前产生。应保持和 SM_ACTIVATED 具有相同的优先级，以保证此消息总在对应的 SM_ACTIVATED 之前被处理。
SM_DEACTIVATED			0x0006

// UI 消息。

// Shell 绘制消息。参考优先级 0xE0 。
//参数类型 shared_ptr<Desktop> 。
//由于绘制对应的桌面。具体行为由派生类定义（例如对于已知的多个桌面，参数可能会被忽略）。
SM_PAINT				0x000F

// 调度消息。

//退出消息。参考优先级 0xFF 。
//参数类型 int 。
//终止进程。通过调用 PostQuitMessage 函数产生。参数被传递作为退出码。
SM_QUIT					0x0012

//任务消息。参考优先级 0x20 。
//参数类型 std::function<void()> 。
//调用参数指定的函数，一般作为后台任务。应注意保持相关对象的有效的生存期，如无法满足应从消息队列中移除。
SM_TASK					0x0016

// Shell 通用交互消息。

// Shell 扩展消息。

//输入消息。参考优先级 0x40 。
//参数类型 InputContent 。
//记录标准输入设备的输入事件。参数为输入上下文，表示按键信息等状态。
SM_INPUT				0x4001

@4 DS 测试项目 YReader
文本阅读器和十六进制阅读器。

@4.1 文本浏览：
支持 txt 、 c 、 cpp 、 ini 等后缀按文本打开。
使用随机载入复杂度为 O(1) 的算法，缓冲时间和文本文件大小无关。

@4.1.1 编码支持：
支持 UTF-8 、 UCS2-LE 、 UCS2-BE 、 UCS-4LE 、 UCS-4BE 编码。
非以上编码按 GBK 打开，可以顺序阅读，但不保证随机读取正常（GBK 无法保证随机定位时一定能选取到字符起始字节）。模拟器上第一次读取 GBK 极慢（可能需要等待数十秒），因为需要在二进制文件中载入编码表，属于正常情况。

@4.1.2 操作说明：
上下换行，左右翻页， LR 调整行距， XY 调整字体 。
 Start 开启自动滚屏（默认 1 行/页），任意其它输入停止。
点击屏幕显示/隐藏显示状态和详细功能按钮的阅读框。
点击阅读框中的进度随机定位。点击 M 按钮打开菜单。点击 I 按钮打开文件和阅读位置信息。点击 R 按钮返回。点击 ←→ 按钮前进/后退（切换最近浏览位置）。
其它功能，如切换字体和背景颜色等，使用菜单（ M 按钮）选择“设置”进入详细设置界面自定义。
